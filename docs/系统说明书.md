## 一、系统概述

梨江中学德育管理平台是一套面向**初中德育全过程数字化管理**的 Web 应用系统，覆盖从**德育处下发任务 → 年级组分配 → 班主任执行 → 家长查看**的完整数据流转链条。

### 1.1 系统目标

- 📋 **德育工作数字化**：替代纸质记录，实现德育数据采集、统计、分析全流程线上化
- 🔔 **多角色协同**：德育处、年级组、班主任、家长四个角色在同一平台协作
- 📊 **数据驱动决策**：通过 AI 预警模型，提前识别需要关注的学生
- 🔔 **家校互联互通**：家长可实时查看学生在校表现，参与学生成长过程

### 1.2 系统特色

| 特色 | 说明 |
|------|------|
| 实时消息推送 | 基于 SSE 技术，新消息/通知即时推送到浏览器，无需刷新 |
| AI 预警模型 | XGBoost 机器学习模型，基于 6 维特征预测学生风险（准确率 76%，召回率 99%）|
| 综合素质评价 | 五维指标体系，支持多角色（班主任/任课教师/家长）评分 |
| 数据可视化大屏 | 德育数据实时展示，支持全校/年级/班级多层级视角 |
| 全角色权限隔离 | 每个角色只能访问授权数据，数据安全有保障 |

### 3.2 消息中心（`/common/messages`、`/common/announcements`）

#### 功能说明
- **实时推送**：新消息到达时，浏览器右下角自动弹出通知（无需刷新页面）
- **消息分类**：系统通知、任务通知、审批通知、家长来信
- **公告管理**：德育处可发布全校/年级/班级公告

#### 操作步骤（查看消息）
1. 点击右上角 🔔 图标
2. 未读消息以**粗体**显示
3. 点击消息可查看详情
4. 回复消息：在详情页底部输入回复内容，点击「发送」

#### 操作步骤（发布公告 — 德育处）
1. 进入「消息中心」→「发布公告」
2. 填写公告标题、内容
3. 选择发布范围（全校/指定年级/指定班级）
4. 点击「发布」

---

### 3.3 学生管理（`/ms/students`、`/grade/students`、`/class/students`）

#### 功能说明
- 按年级/班级查看学生列表
- 查看学生详情（基本信息、成绩、综合素质、违纪记录）
- 支持按姓名/学号搜索

#### 操作步骤（查看学生详情）
1. 进入「学生管理」
2. 点击某学生的「查看」按钮
3. 在详情页可切换查看：基本信息、成绩记录、综合素质、违纪记录、干预记录

---

### 3.4 成绩管理（`/scores/*`）

#### 功能说明
- 班主任/年级组/德育处可录入学生成绩
- 支持按学科、考试类型筛选
- 成绩变更自动触发 AI 分析（如已启用）

#### 操作步骤（录入成绩 — 班主任）
1. 进入「成绩管理」→「录入成绩」
2. 选择班级、学科、考试类型
3. 在表格中填写学生成绩
4. 点击「保存」

> 💡 **提示**：成绩保存后，系统会自动分析是否有需要关注的学生（AI 预警功能）

---

### 3.5 综合素质评价（`/quality/*`）

#### 功能说明
- 五维指标体系：**品德发展**、**学业水平**、**身心健康**、**艺术素养**、**社会实践**
- 支持多角色评分（班主任、任课教师、家长）
- 自动生成班级雷达图

#### 操作步骤（班主任评分）
1. 进入「综合素质」→「我的班级」
2. 选择评价维度（如「品德发展」）
3. 为每位学生打分（1-5 分）
4. 点击「提交」

#### 操作步骤（家长参与评价）
1. 登录家长账号
2. 进入「综合素质」→「评价我的孩子」
3. 按维度打分
4. 点击「提交」

> 📊 **数据说明**：系统会综合多角色评分，生成学生综合素质报告

---

### 3.6 违纪记录管理（`/discipline/*`）

#### 功能说明
- 班主任可记录学生违纪行为
- 支持违纪等级分类：🟡 警告 / 🟠 轻微 / 🔴 重大 / ⚫ 严重
- 违纪记录自动触发**扣分**（综合素质评分）
- 学生/家长可提交申诉

#### 操作步骤（记录违纪 — 班主任）
1. 进入「违纪管理」→「新增记录」
2. 选择学生、违纪类型、违纪等级
3. 填写违纪说明
4. 点击「提交」
5. 系统自动通知德育处和学生家长

#### 操作步骤（处理申诉 — 德育处）
1. 进入「违纪管理」→「申诉处理」
2. 查看申诉详情
3. 选择「采纳」或「驳回」
4. 填写处理意见，点击「提交」

---

### 3.7 请假管理（`/leave/*`）

#### 功能说明
- 家长可提交请假申请
- 班主任/年级组长可审批
- 支持批量审批

#### 操作步骤（提交请假申请 — 家长）
1. 登录家长账号
2. 进入「请假申请」→「新建申请」
3. 填写请假类型、开始时间、结束时间、请假事由
4. 点击「提交」
5. 等待班主任审批（审批结果会通过消息通知）

#### 操作步骤（审批请假 — 班主任）
1. 进入「请假管理」→「待审批」
2. 点击某条申请的「审批」按钮
3. 选择「批准」或「拒绝」
4. 填写审批意见，点击「提交」

---

### 3.8 心理筛查与 AI 预警（`/survey/*`、`/ai-analysis/*`）

#### 功能说明
- **心理筛查**：使用 MSSMHS-55 量表（55 题）对学生进行心理健康筛查
- **AI 预警**：XGBoost 模型基于 6 维特征预测学生风险
  - 准确率（Accuracy）：76%
  - 召回率（Recall）：99%（几乎不漏报高风险学生）
- **干预闭环**：发现风险 → 创建干预记录 → 跟踪干预效果

#### 操作步骤（查看 AI 预警 — 班主任）
1. 进入「AI 预警」→「风险学生列表」
2. 查看风险等级（🟢 低风险 / 🟡 中风险 / 🔴 高风险）
3. 点击某学生可查看详细分析报告
4. 如需干预，点击「创建干预记录」

#### 操作步骤（创建干预记录 — 班主任）
1. 在风险学生详情页，点击「创建干预记录」
2. 填写干预类型、干预内容、计划开始/结束时间
3. 点击「保存」
4. 干预结束后，点击「填写随访记录」更新干预效果

---

## 五、常见问题 FAQ

### 5.1 登录问题

**Q：忘记密码怎么办？**
A：请联系德育处管理员（系统管理员）重置密码。

**Q：登录后页面空白/报错？**
A：请尝试清除浏览器缓存（Ctrl+Shift+Del），或切换到 Microsoft Edge/Chrome 浏览器。

### 5.2 消息通知问题

**Q：为什么收不到实时消息推送？**
A：请检查浏览器是否允许本站点发送通知；如果使用 AdBlock 等广告拦截插件，请将其禁用或将本站点加入白名单。

**Q：消息中心显示有未读消息，但看不到？**
A：尝试刷新页面（F5），或点击消息中心的「全部标记为已读」后重新进入。

### 5.3 数据录入问题

**Q：录入成绩/评分时页面卡住？**
A：请检查是否所有必填项都已填写；如果问题持续，请联系技术支持。

**Q：提交后数据没有保存？**
A：请确认点击了「保存」按钮；如果页面报错，请截图并联系技术支持。

### 5.4 权限问题

**Q：为什么看不到某些功能菜单？**
A：这可能是权限设置问题。请确认您的角色是否正确；如果角色正确但仍看不到，请联系德育处管理员。

**Q：家长能看到其他学生的数据吗？**
A：不能。家长只能查看自己子女的数据，无法访问其他学生的信息。

### 5.5 系统性能问题

**Q：页面加载很慢？**
A：请检查网络连接；如果在学校内网访问仍然很慢，请联系技术支持。

**Q：系统在高峰期（如月考后）变慢？**
A：这是正常现象，系统会在高峰期后自动恢复。德育处正在优化系统性能。

## 六、技术架构简介

> 本章节主要面向学校 IT 管理员和技术支持人员。

### 6.1 系统架构图

```
┌─────────────────────────────────────────────────────┐
│                   浏览器 (用户)                      │
└─────────────────────┬───────────────────────────────┘
                      │ HTTPS
┌─────────────────────▼───────────────────────────────┐
│                  Nginx (反向代理)                    │
│  - 静态资源服务 (CSS/JS/图片)                       │
│  - HTTPS 终止                                       │
│  - 请求转发到 Gunicorn                              │
└─────────────────────┬───────────────────────────────┘
                      │ HTTP (127.0.0.1:5001)
┌─────────────────────▼───────────────────────────────┐
│            Gunicorn (WSGI HTTP 服务器)               │
│  - --workers 2 --threads 8                         │
│  - gthread worker (支持 SSE 长连接)                 │
│  - --timeout 3600 (1 小时)                         │
└─────────────────────┬───────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────┐
│              Flask 应用 (Python 3.13)               │
│  - 24 个蓝图 (Blueprint)                           │
│  - SQLAlchemy ORM                                   │
│  - Jinja2 模板引擎                                  │
│  - SSE 实时消息推送                                 │
└─────────────────────┬───────────────────────────────┘
                      │ SQL
┌─────────────────────▼───────────────────────────────┐
│              MySQL 8.0 (Docker)                     │
│  - 34 张数据表                                      │
│  - 独立容器部署 (grade7-new-db)                    │
│  - 端口：3307 (仅绑定 127.0.0.1)                 │
└─────────────────────────────────────────────────────┘
```
